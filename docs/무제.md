컴포넌트들은 projectId에 종속되지 않게 수정해주고, 내가 다른 스키마들을 줄꺼야 이걸 참고해서
균형있는 component 관련 스키마와 definition 타입을 나에게 제시해줘

## Project

여러 Space를 포함하는 논리적 최상위 단위(정체성 고정)

### Fields
- `id` 
- `headRevisionId` _(현재 활성 리비전)_ 
- `name`
- `createdTime`
- `deletedTime`
### Rules
- `id`는 **불변**
- “현재 상태”는 `headRevisionId`가 가리키는 Revision 스냅샷이 정답
- 과거 보기/퍼블리시/복구는 항상 특정 `revisionId` 기준으로 로드

---

## Space

여러 Page를 포함하는 단위(피그마의 페이지처럼 빈공간) — 워크카피/원본

### Fields
- `id`
- `projectId`
- `name`
- `order`
- `createdTime`
- `deletedTime`
### Rules
- `spaceId`는 **불변**
- 워크카피 편집 가능
- 정확한 재현/퍼블리시는 `SpaceSnapshot(revisionId)` 기준

---

## Page

웹페이지 단위 — 워크카피/원본

### Fields
- `id`
- `projectId`
- `spaceId`
- `rootPageTreeId` _(현재 워크카피 루트)_
- `name`
- `order`
- `createdTime`
- `deletedTime`
### Rules
- `pageId`는 **불변**
- 과거/퍼블리시 렌더는 `PageSnapshot(revisionId).rootPageTreeId` 기준

---

## PageTree

렌더링 되는 노드 단위 — 워크카피/원본

### Fields
- `id`
- `projectId`
- `pageId`
- `componentManifestId`
- `parent: pageTreeId | null`
- `children: pageTreeId[]`
- `order`
- `createdTime`
- `deletedTime`
- `variantId`
- `propsOverride: ComponentManifestProps[] (인스턴스 delta)`_
### Rules
- `pageTreeId`는 **불변**
- 워크카피에서 트리 편집 가능
- 스냅샷 의미를 위해 리비전 생성 시 `PageTreeSnapshot`에 노드 전체 상태 저장

---

## Revision

체크포인트 단위의 스냅샷 메타(스냅샷 본문은 RevisionRepo에 저장)

### Fields
- `id`
- `projectId`
- `parentRevisionId` _(선형 히스토리용: 직전 리비전)_
- `restoredFromRevisionId` _(optional: 복구 대상 리비전)_
- `reason` _(manual / componentUpdateApplied / restored 등)_
- `designSnapshotToken`
- `componentManifestToken`
- `createdBy`
- `createdTime`
- `deletedTime`
### Rules
- **immutable**
- 히스토리는 **선형**: `parentRevisionId`
- 복구는 항상 새 리비전 생성:
	- `reason = restored`
	- `restoredFromRevisionId = <targetRevisionId>`
	- `parentRevisionId = <restore 직전 headRevisionId>`
- SDK/퍼블리셔 입력은 `revisionId` 단일

---

## SpaceSnapshot

Space의 리비전 스냅샷

### Fields
- `revisionId`
- `id` _(=spaceId)_
- `projectId`
- `name`
- `order`
- `pages: pageId[]` _(스냅샷 당시 구성/순서; 삭제된 pageId도 포함)_
- `createdTime`
- `deletedTime`
### Rules
- `deletedTime`이 있더라도 스냅샷에 포함
- `pages[]`는 **삭제된 pageId도 포함**하며, 스냅샷 당시 배열을 그대로 보존

---

## PageSnapshot

Page의 리비전 스냅샷

### Fields
- `revisionId`
- `id` _(=pageId)_
- `projectId`
- `spaceId`
- `name`
- `order`
- `rootPageTreeId` _(삭제된 노드일 수도 있음 — 그대로 보존)_
- `createdTime`
- `deletedTime`
### Rules
- `deletedTime`이 있더라도 스냅샷에 포함
- 트리 구성은 `rootPageTreeId` + `PageTreeSnapshot`으로 재구성

---

## PageTreeSnapshot

PageTree 노드의 리비전 스냅샷(트리 전체 상태 저장)

### Fields
- `revisionId`
- `id` _(=pageTreeId)_
- `projectId`
- `pageId`
- `componentManifestId`
- `variantId`
- `parent: pageTreeId | null`
- `children: pageTreeId[]` _(삭제된 child도 포함)_
- `propsOverride: ComponentManifestProps[]`
- `order`
- `createdTime`
- `deletedTime`
### Rules
- `deletedTime`이 있더라도 스냅샷에 포함
- `children[]`는 **삭제된 child id도 포함**하며, 스냅샷 당시 배열을 그대로 보존
- 스냅샷의 의미를 위해 `parent/children/order/style` 등 **노드 전체 상태** 저장
